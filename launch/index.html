Canonical URL launch contract

What an agent should do after the user pastes the Sula launch URL.

This page is the human-readable contract behind the short launch sentence. It is designed to remove startup pain: read the page, resolve the launcher, classify the target project, then route to onboarding or existing-consumer review.

Behavior Contract

The default launch sequence is fixed.

  1. Read this page and the machine-readable descriptor at /sula.json.
  2. Do not search npm packages, global binaries, or guessed local aliases.
  3. Resolve the launcher in this order:
    a. use vendored scripts/sula.py if the target project already includes it
    b. use an explicit local Sula source checkout if one is provided
    c. otherwise fetch the canonical public source only after /sula.json declares a published public repository
  4. If .sula/project.toml exists, treat the project as an existing consumer.
  5. If the project is not yet adopted, run guided onboarding instead of low-level adoption by default.
  6. Report the Sula source that was used, the actions taken, the managed boundary, and the next commands.

New Projects

Route through onboarding.

python3 bootstrap.py --project-root /path/to/project

or

python3 bootstrap.py --project-root /path/to/project --accept-suggested --approve
  • Use `onboard` as the preferred first-touch flow.
  • Ask only the minimum missing questions.
  • Explain what Sula will manage and where artifacts will route.

Existing Consumers

Review instead of readopting.

doctor --strict
sync --dry-run
status --json
  • Do not rerun first-time adoption over an existing consumer.
  • Detect drift, preview sync impact, and explain next steps.
  • Keep the user on the same managed/scaffold contract they already adopted.

No Guessing Rule

The site must be enough to resolve startup.

  • The site should tell agents whether the canonical public source is published yet.
  • The site should expose a launcher that can bootstrap local work without guessing a pre-public repository.
  • The site should make it explicit that package-name guessing is a protocol violation.
