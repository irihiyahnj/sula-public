# OKOKTOTO v5 Change Records

This file is the index for non-trivial project changes.

## Purpose

Git records code differences. Change records explain:

- why the change happened
- how the decision was made
- what was verified
- how rollback works
- what data or operational side-effects exist

## Rules

- keep this file short and index-oriented
- put detailed records in `docs/change-records/`
- use release records for rollout history that goes beyond a single code change
- use incident records for outages, broken flows, or recovery work
