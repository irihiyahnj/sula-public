This file records approved exceptions to the highest architecture rule.

## Usage Rules

1. Default state is no exception.
2. Exceptions must be minimal and explicitly approved.
3. Every exception must have a matching change record.
4. If the exception can be retired later, mark it retired and record the replacement path.
5. Use a change-record link in the `Reference` column, for example `../change-records/YYYY-MM-DD-short-topic.md`.
